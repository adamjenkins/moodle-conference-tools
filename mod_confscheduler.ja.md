# Conference Scheduler（スケジュール）— ユーザーマニュアル

**アクティビティタイプ:** `mod_confscheduler`
**Conference Tools スイートの一部です** ([概要](README.md))

Conference Scheduler は、[Conference Program](mod_confprogram.ja.md) で
採択された発表を、ドラッグ＆ドロップ操作で「時間 × 会場」形式のブロッ
クスケジュールに配置するためのアクティビティです。完成したスケジュー
ルは参加者向けに読み取り専用のタイムテーブルとして公開されます。

> 画面上の項目名やボタンの表示は英語のままです（本マニュアルは操作の
> 説明を日本語で行うものです）。

## 主催者向け

### アクティビティの追加

1. コースに **Conference Scheduler** アクティビティを追加します。
2. 同じコース内の **Conference Program** アクティビティにリンクしま
   す。ここで採択された発表がスケジュール対象になります。
3. **General（一般）** セクションで**開催開始日・終了日**を設定しま
   す。これらは**必須項目**です——選択できる日付（「All days」表示を
   含む）、自動スケジューラーの初期の時間範囲、そして開始前・終了後の
   時間をグレー表示にしてブロックをドラッグできないようにする挙動の
   すべてが、この日付範囲を基準にしています。
4. 保存します。

**SnapGap の最小間隔**（後述）はここでは設定しません——Edit mode を
オンにすると、スケジュール画面の上部にクイック設定として表示されます。

### 編集モードをオンにする

Conference Scheduler は、**Moodle 本体のコース「Edit mode」スイッチ**
をそのまま使用します。これは自分の名前の横、ページ上部に常に表示され
ているスイッチです。編集がオフの間は、あなたを含め誰にでも参加者と同
じ読み取り専用のタイムテーブルが表示されます。変更を行うにはオンにし
てください。作業が終わったら再度オフにすると、同じ読み取り専用の表示
に戻ります。このプラグイン専用の別のトグルはありません——コース内の他
の項目を編集する際に使っている Edit mode と全く同じ操作です。

Edit mode がオンの間は、会場管理、ドラッグ＆ドロップによるスケジュー
リング、自動スケジューラーを利用できます。ブロックをドラッグしている
間は、ドロップした場合に実際に配置される位置（後述の SnapGap による
自動調整後の位置を含む）がハイライト表示され、離す前に確認できます。

### 会場（ルーム）の管理

Edit mode がオンの状態で **Add room** を選ぶと、新しい列（会場）を作
成できます。各会場について次を設定できます。

- **名前**
- **色**（任意）— 列見出しの背景色として使われます。見出しの文字色
  は、選んだ背景色に対して読みやすくなるよう自動的に黒／白に切り替わ
  るため、コントラストについて別途気にする必要はありません。
- **定員**（任意）— 会場の最大収容人数です。設定すると列見出しに表示
  されます。空欄のままにすると定員なし（無制限）として扱われます。

会場の見出しをドラッグすると列の並び替えができます。会場はいつでも編
集・削除できますが、まだ発表が割り当てられている会場は、先にその発表
を移動または削除しないと削除できません。

会場に定員を設定している場合、その会場に配置された発表のお気に入り数
が定員を超えると、（Edit mode の場合のみ）そのブロックが**定員超過の
可能性あり**として強調表示されます。これはあくまで情報提供であり、配
置を制限するものではありません——立ち見や、その時間帯に他に空いてい
る大きな会場がない場合など、意図的に定員を超えて配置することも主催者
の判断として認められています。

### 発表のスケジューリング

左側の **Unscheduled（未割り当て）** パネルには、まだグリッドに配置さ
れていない採択済みの発表が、タイトル・発表者・トラックとともに一覧表
示されます。Edit mode がオンの状態で以下の操作ができます。

- Unscheduled パネルからカードを**ドラッグ**して会場・時間帯に配置す
  る（スケジューリング）。このとき、Conference Submissions（発表登
  録）側でその発表に設定されている**発表タイプ**の所要時間がそのまま
  使われます。そのためライトニングトークとワークショップのように長さ
  が異なる発表でも、それぞれ正しい長さで配置され、あとから一つひとつ
  手作業でリサイズする必要がありません。
- 既に配置済みのブロックを**ドラッグ**して別の会場・時間に移動する
  （再スケジューリング）。
- ブロック下端の小さな**ハンドル**を**ドラッグ**して所要時間（長さ）
  を変更する——これは常にその1つのブロックだけに対する上書きで、発表
  タイプ側の設定や他のブロックには影響しません。
- ブロックの**★（スター）**をクリックしてお気に入り登録する（これは
  Conference Program や参加者が使うお気に入りと同じものです。詳しく
  は [Conference Program のマニュアル](mod_confprogram.ja.md) を参照
  してください）。
- ブロックの**トラックのピル（バッジ）**をクリックすると、Conference
  Program 側でそのトラックに絞り込んだ一覧に直接移動します。

### 日付と開催期間

スケジュール上部の**日付（Day）**プルダウンには、開催開始日・終了日
の範囲内のすべての日（まだ何も配置されていない日も選べます）に加え
て、**All days（全日程）**が表示されます。全日程を選ぶと、すべての日
を1日ずつのテーブルとしてまとめて同じページに表示でき、イベント全体
をざっと確認したいときに便利です。編集（ブロックのドラッグ）は1日ず
つの表示でのみ行えます。何かを再スケジュールしたい場合は、まず「All
days」以外の表示に切り替えてください。

開催開始日・終了日の範囲外の時間帯は斜線のグレー表示になり、そこに発
表をドロップすることはできません。グレー表示の領域にブロックをドラッ
グしようとすると、SnapGap が別のブロックに近すぎるときにブロックを弾
き返すのと同様に、自動的に最も近い有効な時間へ戻されます（詳しくは後
述）。

### SnapGap（間隔の自動調整）

SnapGap は、同じ会場で連続して配置された発表の間に最低限の間隔を確保
する機能です。発表者や参加者が会場を移動する時間を確保するのに役立ち
ます。間隔（分単位）は、Edit mode がオンのときにスケジュール画面上部
に表示される**SnapGap minimum gap** コントロールで設定します。変更は
即座に反映されます。

**SnapGap はドラッグ操作をエラーで拒否しません。** 間隔の条件を満たさ
ない場所（あるいは実際に重なってしまう場所）にブロックをドロップする
と、自動的に**条件を満たす最も近い位置へスナップ（吸着）**します。必
要な分だけブロックがずれて間隔が確保されるか、重ならない時間に戻され
ます。常に有効な配置に落ち着くため、何度も試行錯誤してドラッグをやり
直す必要はありません。

### 行の高さ

SnapGap のコントロールのすぐ隣にあるのが、**Row height (pixels per
hour)**（行の高さ）です。スケジュール上で1時間分がどれくらいの高さで
表示されるかを調整します。デフォルト値でほとんどの会議には十分です
が、短い発表でタイトルや発表者名が窮屈にならないよう表示スペースを増
やしたい場合は高さを大きくし、1日分をスクロールせずに画面へ収めたい
場合は小さくしてください。変更は即座に反映され、全員（主催者・参加者
の両方——参加者向けの閲覧専用画面も同じ設定を使うため）に適用されま
す。すでに配置済みのブロックの位置や長さが変わることはなく、見た目の
密度だけが変わります。

### 表示開始時刻・表示終了時刻

デフォルトでは、スケジュールグリッドはその日に実際に配置されている内
容から表示範囲を決めます。何も配置されていない状態でも常に固定の日次
表示範囲（例えば08:00〜18:00）を表示させたい場合は、行の高さコントロ
ールの隣にある**自動**のチェックを外し、**表示開始時刻**と**表示終了
時刻**を設定してください。

**この設定は、日程セレクターで表示している日に適用されます。**開催日
ごとに終了時刻が異なることはよくあるため、日ごとに別々に設定できます。
特定の日を選べばその日だけに適用され、**すべての日**を選べば個別設定
のない日すべてに使われる既定値を設定できます。コントロールの隣のラベ
ル（**適用対象: …**）に、どちらを編集しているかが常に表示されます。特
定の日で**自動**をチェックしたままにすると「独自の範囲なし——すべての
日の既定値を使用」となり、そこで範囲を設定するとその日だけ上書きされ
ます。

ある日の表示範囲外に何かが配置された場合（早朝の準備セッションなど）
でも、内容は省略されず全体が表示されます——グリッドが必要な分だけ自
動的に広がり、範囲外の部分は開催期間の開始・終了日時の範囲外と同じよ
うにグレー表示され、通常の表示範囲外であることが一目でわかります。

### スパンブロック（全体会・昼食・休憩など）

投稿された発表ではないもの——全体会、昼食休憩、複数会場にまたがる基調
講演など——には **Add span block** を使用します。ラベル、色、時間帯、
またがる会場の範囲を指定します。会場と同様に、選んだ色に対して文字色
のコントラストが自動的に調整されます。

スパンブロックは**作成後も編集**できます。Edit mode がオンの状態で既
存のスパンブロックをクリックすると、内容が入力済みの同じフォームが開
き、ラベル・色・時間・会場の範囲を変更できます。

### 自動スケジューラーの実行

規模の大きいプログラムでは、すべてのブロックを手作業で配置するのは大
変です。Edit mode がオンの状態で **Run autoscheduler** を選び、以下を
設定します。

- 配置対象とする**時間帯（ウィンドウ）**——開催開始日・終了日があらか
  じめ入力されているため、通常は変更不要です。
- その時間帯に**既存のスケジュールを先にクリアするかどうか**（デフォ
  ルトはオフ。オフの場合、自動スケジューラーは既に配置済みの内容を避
  けて配置します）
- **希望日程を無視するかどうか**（デフォルトはオフ。詳しくは後述）

各発表は、Conference Submissions 側でその発表に設定されている発表タ
イプの所要時間で配置されます（手動でドラッグする場合と同じです。上記
参照）。そのため所要時間について別途設定する項目はありません。

自動スケジューラーは、可能な限り同じ**トラック**の発表を同じ会場にま
とめて配置することを優先し、設定した SnapGap の間隔を守り、実行のた
びに配置順をランダム化します（そのため、同じ条件で再実行しても毎回ま
ったく同じ配置になるとは限りません）。何件配置できたか、そして時間帯
内にどうしても配置できなかった発表があればその理由も、結果として表示
されます。

**希望日程について**：発表者が希望する開催日を登録している場合
（Conference Submissions の「希望日程を提示する」機能）、自動スケ
ジューラーは既定でその発表を希望日にのみ配置します。希望日に空きがな
い場合は、発表者が除外した日に移す代わりに、未配置のまま「配置できま
せんでした」として報告します。実行前に**「希望日程を無視する」**に
チェックを入れるとこれを緩和できます。まず希望日への配置を試み、空き
がなければ他の日にフォールバックし、未配置のまま残しません。希望外の
日に配置された発表は——このような実行によるものでも手動ドラッグによ
るものでも——Edit mode 中は琥珀色の枠とハッチングで注意を促します
（Edit mode をオフにすると他のブロックと同じ見た目になります）。

### 通知

Conference Submissions や Conference Program とは異なり、ここでのスケ
ジュール通知は**自動送信されません**——スケジュールは作成中に何度も組
み直されることが多く、ドラッグのたびに発表者へメールを送ると煩わしい
ためです。代わりに、ツールバー（Edit mode オン時）に**通知を送信**ボ
タンが表示され、前回の通知以降にスケジュール情報が変更された発表の件
数がリアルタイムで表示されます。準備ができたらクリックすると、その発
表者にのみメールが送られます（変更のない発表者はスキップされるため、
後からさらに変更を加えたあとで再度クリックしても、影響のなかった発表
者に重複して通知が届くことはありません）。

**「Manage notifications」** を開くと、この通知の件名・本文をカスタマ
イズできるほか、**「Enable notifications」** でこのアクティビティの通
知を完全に無効化できます。無効化されている間も、通知待ちの件数はスケ
ジュール変更に応じて更新され続けますが、**通知を送信**をクリックして
も何も送信されません。ただし、内容が失われることはありません——通知
を再度有効にしてもう一度ボタンをクリックすれば、無効化されていた間に
たまった分がまとめて送信されます。使用できるプレースホルダーには
`[[fullname]]`、`[[submissiontitle]]`、`[[coursename]]`、
`[[roomnames]]`、`[[starttime]]`、`[[endtime]]` があります。

## 参加者向け

（デフォルトの状態である）コースの編集がオフの状態でアクティビティを
開くと、公開済みの読み取り専用スケジュールが表示されます。

- **日付選択**で、1日ごとにスケジュールを切り替えて表示できます。
  **All days（全日程）**を選ぶと、すべての日のスケジュールを1日ずつの
  テーブルとして同じページにまとめて表示できます。
- 配置されているブロックをクリックすると、Conference Program 側のそ
  の発表の詳細ページが開きます。
- **★（スター）** をクリックすると **My timetable（マイタイムテーブ
  ル）** に追加され、**My timetable** トグルを使うと、自分がお気に入
  り登録したセッションだけがハイライトされ、それ以外はグレーアウトさ
  れます。混み合ったスケジュールの中から自分のセッションをすぐに見つ
  けるのに便利です。
- お気に入りを登録したら、My timetable トグルの隣にある**マイタイム
  テーブルをエクスポート (.ics)** をクリックすると、カレンダーファイ
  ルとしてダウンロードできます。Google カレンダー、Outlook、Apple カ
  レンダーなど、お使いのカレンダーアプリに取り込めます。
- **トラックのピル（バッジ）**をクリックすると、そのトラックの他の発
  表も確認できます。

## スケジュールの印刷

（読み取り専用画面に表示される）**Print** 設定では、**カラー**または
**白黒**を選択できます。白黒モードでは会場の色分けを取り除き、罫線と
テキストのみのシンプルな表示になるため、モノクロプリンターやコピー機
でもきれいに印刷できます。この切り替えは印刷時だけでなく、画面上にも
即座に反映されるようになったため、印刷する前にプレビューできます。

用紙サイズと向きは、このページでは設定しなくなりました。代わりにブラ
ウザ自体の印刷ダイアログで選択してください。多くの場合、こちらの方が
（A2を含む）選択肢が多く、拡大縮小もこのページの固定設定より正確に
調整できます。

## よくある質問

**管理者／教師なのにドラッグ＆ドロップのグリッドが表示されません**
ページ上部、自分の名前の横にある Moodle の **Edit mode** スイッチをオ
ンにしてください。これは管理者を含めて誰に対してもデフォルトではオフ
になっています。コース内の他の項目を編集する際に使うのと同じスイッチ
で、Conference Scheduler 専用の別のトグルはありません。

**ドラッグしたブロックが、ドロップした場所と違う位置に移動しました**
これは SnapGap が自動的に、最小間隔を満たす（または重なりを避ける）最
も近い位置へブロックを調整した結果です。単純にドロップをエラーで拒否
することはありません。

**端末のダークモードに連動した配色にできますか？**
現在、ダークモード対応は一時的に無効化されており、将来のアップデート
で再度対応する可能性があります。現状はスケジュールは常に通常のライト
モードの外観で表示されます。

**設定した開催開始日・終了日は、スケジュールの制約として使われます
か？**
はい。開催開始日・終了日は必須項目で、その範囲外の時間帯はグレー表示
され、ブロックを配置できません。グレー表示の領域にブロックをドラッグ
しようとすると、SnapGap と同様に最も近い有効な時間へ自動的に戻されま
す。詳しくは前述の「日付と開催期間」を参照してください。

**発表を再スケジュールしても発表者にメールが届きません**
このアクティビティのスケジュール通知は自動送信されません。準備ができ
たらツールバーの**通知を送信**をクリックしてください。前回クリックし
たとき以降にスケジュール情報が変更された発表者にのみメールが送られま
す。

**「定員超過」の強調表示があると、その会場にはもう配置できませんか？**
いいえ、これはあくまで情報提供です。すでにお気に入り数が定員を超えて
いる会場にも、引き続き発表を配置できます。立ち見や、その時間帯に他に
空いている会場がない場合など、意図的に定員を超えて配置することもあり
ます。

**マイタイムテーブルをエクスポートしたのに、ファイルの中身が空です**
まだセッションにスターを付けていないか、スターを付けたセッションが現
在スケジュールされていない可能性があります。まず **My timetable** で
少なくとも1つセッションにスターを付けてから、再度エクスポートしてく
ださい。
