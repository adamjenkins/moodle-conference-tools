# Conference Program（審査・プログラム）— ユーザーマニュアル

**アクティビティタイプ:** `mod_confprogram`
**Conference Tools スイートの一部です** ([概要](README.md))

Conference Program は、[Conference Submissions](mod_confsubmissions.ja.md)
で集められた投稿の審査ワークフローを管理し、審査終了後は採択された発
表の一覧を参加者に公開します。このアクティビティは常にどちらかの
**フェーズ（段階）** で動作します。

- **Review phase（審査フェーズ）** — 審査者が投稿を採点し、主催者が採
  択・不採択・補欠・再投稿依頼などの決定を記録します。参加者にはまだ
  何も公開されません。
- **Display phase（公開フェーズ）** — 採択された発表の一覧が参加者向
  けに公開され、閲覧・絞り込み・お気に入り登録ができるようになりま
  す。審査フェーズの画面は読み取り専用になります。

> 画面上の項目名やボタンの表示は英語のままです（本マニュアルは操作の
> 説明を日本語で行うものです）。

## 主催者向け

### アクティビティの追加

1. コースに **Conference Program** アクティビティを追加します。
2. 設定画面で、同じコース内のどの **Conference Submissions** アクティ
   ビティから投稿を取得するかを選択します。このリンクは必須です。リ
   ンクがなければ審査する投稿がありません。
3. 保存します。アクティビティは **Review phase（審査フェーズ）** の状
   態で開始されます。

現在のフェーズやフェーズ切り替えボタンは、編集モードをオンにしている
主催者にのみ表示されます。参加者や審査者がページ上でフェーズに関する
表示を目にすることはありません。

### ブラインド審査

アクティビティの設定で **「Blind review」** をオンにすると、審査ワー
クフロー全体を通じて、投稿者と審査者の身元が互いに非表示になります。
これは発表者一覧だけでなく、投稿にある**主催者定義の独自項目（カスタ
ムフィールド）すべて**も対象です（「著者紹介」のような自由記述項目
も、氏名と同じように身元を特定しうるためです）。この設定は画面上の表
示にのみ影響し、保存されているデータ自体を制限するものではありませ
ん。また、インスタンスが公開フェーズに切り替わった後は効果を持ちませ
ん（公開フェーズでは、決定後の身元は公開されるのが前提のためです）。

既定では、**Manager** ロールのみがブラインド審査を回避（身元を見るこ
とが）できます。他のロール（例えば査読も行う共同主催者など）にも審査
中に身元を見せたい場合は、そのロールに *「実際の投稿者・審査者の身元
を表示する」* 権限を、アクティビティまたはコースレベルで明示的に付与
してください。編集権限を持つロールであれば自動的に身元も見られる、と
想定しないでください——編集権限と身元表示の回避は意図的に別の権限と
して分けられています。

### 審査者の割り当て

**「Assign reviewers」** を開きます。以下のことができます。

- 個々の審査者を個々の投稿に割り当てる。
- コース内の既存の**グループ**（Groups）機能を利用して、グループ単位
  で審査者をまとめて割り当てる（別途「審査者グループ」という概念は使
  いません）。
- 投稿一覧をトラックで絞り込んでから一括割り当てを行う（例：「技術」
  トラックの投稿はすべて技術系の審査者へ）。
- Decisions 画面の **Start a new round**（新しいラウンドを開始）リンク
  から来た場合は、次のラウンドの審査者待ちで **Resubmit（再投稿依頼）**
  決定が付いた投稿だけにあらかじめ絞り込まれた状態で表示されるため、
  そのままグループへの一括割り当てを行えます。
- 審査者1人あたりの**最大審査件数**を、アクティビティ全体のデフォル
  ト値として、または審査者ごとに個別に設定し、特定の審査者に負担が集
  中しないようにする。

### ルーブリックの設定

審査の採点には Moodle 標準のルーブリック採点機能を使用します。審査画
面から開くことができます。評価基準と評価レベルを一度定義しておけば、
Moodle 標準のルーブリックテンプレート共有機能を通じて、他のトラック
や今後のアクティビティでも同じテンプレートを再利用できます。

### 「審査対象外（unvetted）」フラグ

招待講演やパネルディスカッションなど、主催者が既に直接調整済みで査読
の必要がない投稿もあります。このような投稿は決定画面から
**unvetted（審査対象外）** に設定できます。審査対象外の投稿は次のよう
に扱われます。

- 審査者割り当て画面や「自分の審査キュー」画面には表示されません。
- 主催者が直接（通常は「Accept」として）決定を記録できます。
- 気が変わればいつでも解除できます。

### 決定の記録

投稿に十分な数の審査が集まったら、**Decisions（決定）** レポートから決
定を記録します。このレポートは審査対象外でないすべての投稿を一覧にし
た表で、各行にトラック・現在の審査ラウンド・直近の決定（あれば）・各
審査者のスコアが表示されます。表の上にある**トラック**フィルターと
**決定ステータス**（未決定 / 採択 / 不採択 / 再投稿依頼 / 補欠）フィル
ターで絞り込めます。

1件ずつ決定する場合は、各行のドロップダウンと **Save decision（決定を
保存）** ボタンを使います。複数件をまとめて処理したい場合は、対象の行
にチェックを入れ（または見出し行のチェックボックスで表示中のすべての
行を選択し）、表の上のツールバーから **Accept（採択）**・
**Reject（不採択）**・**Waitlist（補欠）**・**Resubmit（再投稿依頼）**
のいずれかを選んで適用ボタンをクリックします。確認ダイアログに選んだ
決定内容と対象件数が表示されるので、確認してから確定してください。ブ
ラインド審査（後述）を有効にしている場合、発表者に表示される審査コメ
ントから審査者の氏名が分かることはありません。

リンク先の Conference Submissions アクティビティで投稿を編集できる権
限（編集教師・管理者）を持っている場合、各行のタイトルにも「Edit」リ
ンクが表示されます。この画面を離れることなく、トラックなどの詳細を修
正できます。保存（またはキャンセル）すると、その時点で適用していたト
ラック・決定ステータスのフィルターを保持したまま Decision report に戻
ります。

**Resubmit** を選ぶと、そのコメント付きで発表者の投稿（Conference
Submissions 側）に差し戻され、発表者が内容を修正し、その後再度審査で
きるようになります。Accept／Reject／Waitlist とは異なり、Resubmit の通
知は公開フェーズを待たず**即座に**メール送信されます（発表者がコメン
トを確認して再投稿できるページへのリンク付き）——すぐに対応してもら
うことが目的のためです。即座に送信されるため、各行の **Save decision**
ボタンでも一括適用ツールバーでも、記録・送信前に決定内容を示す確認ダ
イアログが表示され、準備ができていなければキャンセルできます。
**Resubmit** の決定が1件でも付くと、表の上に
**Start a new round**（新しいラウンドを開始）リンクが表示されます。こ
のリンクをたどると、次のラウンドの審査者待ちの再投稿投稿だけにあらか
じめ絞り込まれた **Assign reviewers**（上記「審査者の割り当て」を参
照）画面に直接移動でき、1件ずつ個別に対応する必要はありません。

### 公開フェーズへの切り替え

審査が終わったら、（編集モード時のみ表示される）現在のフェーズ表示の
隣にあるフェーズ切り替えボタンを使って **Display phase（公開フェー
ズ）** に切り替えます。これはアクティビティ全体を一度に切り替える操作
です。参加者に採択一覧を公開する準備が整ってから実行してください。

### 参加者に表示する内容の制御

**「Display field settings」** を開くと、投稿の各項目（タイトル・アブ
ストラクト・トラック・発表者・Conference Submissions 側で追加した独
自項目など）ごとに、以下のどちらに表示するかを設定できます。

- **一覧表示**（参加者が閲覧するコンパクトな表形式の画面）
- **詳細ポップアップ**（投稿をクリックすると開く詳細情報画面）

これにより、一覧はコンパクトに（例：タイトル・トラック・発表者・時
間／会場のみ）保ちつつ、詳細ポップアップでは完全なアブストラクトや追
加項目もすべて確認できるようにする、といった使い分けが可能です。

### 通知

**新規に追加したアクティビティでは通知は初期状態でオフ**になっていま
す。決定をメール通知したい場合は、下記の **「Enable notifications」**
をオンにしてください。

Accept・Reject・Waitlist の決定は、記録した瞬間に自動送信されることは
なく、公開フェーズへの切り替え時にも自動送信されなくなりました。代わ
りに、準備ができたら **「Send pending notifications」**（このページに
送信待ちの件数がリアルタイムで表示されます）をクリックすると、まだ送
信していない決定を持つすべての発表者へメールが送られます。これによ
り、発表者が公開前に結果を知ってしまうことを防ぎつつ、フェーズ切り替
えのタイミングに縛られず、主催者の都合の良いタイミングで送信できま
す。**「Pending notifications」** を開くと、送信待ちの内容（投稿・決
定・宛先・待機時間）を確認でき、送信したくないものは **「Dismiss」**
できます——Dismiss してもメールは一切送信されず、決定そのものも変更
されません。ある投稿について送信待ちになるのは常に最新の決定のみで
す。そのため、例えば同じ投稿に Waitlist の後で Accept が記録されて
も、メールが2通届くことはありません。

**Resubmit** の決定だけは例外で、フェーズに関わらず常に即座にメール送
信され（詳しくは上記「決定の記録」を参照）、送信待ちキューには表示さ
れません。

各決定タイプ（**Accept**・**Reject**・**Waitlist**・**Resubmit**）はそ
れぞれ独自の件名・本文テンプレートを持ちます——**「Manage
notifications」** を開き、タブを切り替えてそれぞれをカスタマイズして
ください。標準のデフォルト内容は、その結果に応じた次の行動を発表者に
示唆します（例：Accept は早めのチケット確保を促し、Resubmit はコメン
トを確認して修正できるページへのリンクを含みます）。
**「Enable notifications」** でこのアクティビティの通知を完全に無効化
できます。後から通知を再度有効にした場合も、無効化されていた間に記録
された決定はきちんと送信されます（消えてしまうことはなく、送信が遅れ
るだけです）。使用できるプレースホルダーには `[[fullname]]`、
`[[submissiontitle]]`、`[[coursename]]`、`[[decision]]` があり、
Resubmit のテンプレートに限り `[[feedbackurl]]` も使用できます。

## 審査者向け

**「My review queue」** を開くと、自分に割り当てられた投稿が表示され
ます。各投稿について、次の手順で審査します。

1. 審査画面を開いて投稿内容を確認します（ブラインド審査が有効な場合、
   発表者一覧と独自項目（カスタムフィールド）はすべて非表示になり、代
   わりにその旨の案内が表示されます。タイトル・トラック・アブストラク
   トは常に表示されます——ブラインド審査が隠すのは「誰が」であって
   「何を」ではないためです）。
2. 主催者が設定したルーブリックに沿って採点します。
3. 審査を提出します。審査ラウンドの状況によっては、主催者が最終決定
   を記録するまで審査内容を修正できる場合があります。

投稿が後で**再投稿（resubmission）**の対象になった場合、新しいラウン
ドで修正版を再度審査するよう依頼されることがあります。Decisions 画面
にはラウンドごとの履歴が保存されます。

## 発表者向け（再投稿）

投稿に **Resubmit（再投稿依頼）** の決定が下された場合、Conference
Submissions 側で自分の投稿を開いて編集してください。決定に添えられた
審査コメント（ブラインド審査が有効な場合は審査者名は非表示）が編集
フォームと合わせて表示されるので、何を修正すべきか確認できます。変更
を保存すると、投稿は新たな審査ラウンドの対象になります。

## 参加者向け

主催者が **Display phase（公開フェーズ）** に切り替えると、アクティビ
ティを開いて**採択された発表**を閲覧できるようになります。

- **日付選択**を使って、特定の開催日の内容にすばやく切り替えられま
  す（初期表示は本日、複数日開催の場合は本日が範囲外なら初日になりま
  す）。**All days（全日程）**を選ぶと、すべての日の内容を列幅の揃った
  単一の表として表示し、日付ごとに区切り行が挿入されます。スマート
  フォンなど狭い画面では、発表ごとに2行のレイアウトへ折りたたまれ、読
  みやすくなります。
- 発表のタイトルをクリックすると、完全なアブストラクトや主催者が表示
  対象に選んだ追加項目を含む詳細ポップアップが開きます。
- **★（スター）** アイコンをクリックすると **My favourites（お気に入
  り）** に追加され、**Favourites only** フィルターで自分がスターを付
  けたセッションだけを絞り込んで表示できます。このお気に入りは、コー
  スに Conference Scheduler アクティビティがある場合、そちらの「my
  timetable」のハイライトと自動的に同期します。どちらか一方でスター
  を付ければ、両方に反映されます。
- コースに Conference Scheduler アクティビティもある場合、スケジュー
  ルが公開されていれば各行に開催時間と会場が表示されます。

## よくある質問

**採択された発表の一覧がまだ見られません**
主催者がまだ Display phase（公開フェーズ）に切り替えていない可能性が
あります。しばらく待つか、主催者に直接確認してください。

**審査者ですが、特定の投稿が自分の審査キューに表示されません**
まだ自分に割り当てられていないか、その投稿が **unvetted（審査対象
外）** に設定されている可能性があります。

**ここでスターを付けると、スケジュール側にも影響しますか？その逆は？**
はい、意図的にそうなっています。「お気に入り」は Conference Program
の一覧と Conference Scheduler の「my timetable」ハイライトの両方で共
有される、同じ1つの状態です。そのため、どちらか一方でスターを付ける
だけで済みます。

**判定結果のメールは、正確にはいつ届きますか？**
Accept・Reject・Waitlist の場合は、主催者が **「Send pending
notifications」** をクリックした時です——決定が記録された時点でも、
フェーズが切り替わった時点でもありません。それまでは送信待ちとして
キューに入り（**「Pending notifications」** ページで確認・取り消し可
能）ます（主催者がこのアクティビティの通知を完全に無効化していない限
り）。**Resubmit** の決定だけは異なり、常に即座にメール送信されます
——発表者にコメントを確認しすぐ対応してもらうためです。
