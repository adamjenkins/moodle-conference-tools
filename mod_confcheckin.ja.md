# Conference Check-in（受付・チェックイン）— ユーザーマニュアル

**アクティビティタイプ:** `mod_confcheckin`
**Conference Tools スイートの一部です** ([概要](README.md))

> **現在の開発状況について:** 本マニュアルに記載されている機能はすべ
> て実装済みです。チケット種別（グループ・登録方法による自動付与を含
> む）・定員・参加資格の判定機能、決済／プロモコードによる購入フロー、
> QRコード付きの印刷用名札・チケット・領収書・証明書のテンプレート機
> 能、受付でのチェックインスキャナー（ブラウザおよび Moodle アプリ）、
> チェックイン後の証明書発行、すべて利用可能です。

Conference Check-in は、参加チケットの販売・発行、QRコード付き名札の
印刷、当日受付でのチェックイン記録、参加証明書の発行を行うアクティビ
ティです。ここでいう「バッジ（badge）」とは、常に**参加者が身につける
印刷用の名札**を指しており、Moodle 本体の Open Badges（達成バッジ）機
能とは一切関係ありません。

> 画面上の項目名やボタンの表示は英語のままです（本マニュアルは操作の
> 説明を日本語で行うものです）。

## 主催者向け

### アクティビティの追加

1. コースに **Conference Check-in** アクティビティを追加します。
2. 発表者限定のチケット種別を提供したい場合は、同じコース内の
   **Conference Program** アクティビティにリンクします（このリンクは
   任意です。発表者限定チケットを使わない場合は設定不要です）。
3. 保存します。

### チケット種別の設定

**「Manage ticket types」** を開き、参加者が購入・取得できるチケット
の種類を定義します。各チケット種別について次を設定します。

- **名前**（例：「フルカンファレンス」「学生」「発表者」「2日目のみ」
  など）
- **価格と通貨** — 完全に無料のチケットにしたい場合は価格を0に設定し
  ます。価格が0のチケットには決済手続きは発生しません。
- **定員**（任意）— そのチケット種別の発行上限数です。空欄のままにす
  ると無制限になります。多数のユーザーが同時に最後の1枚を購入しよう
  とした場合でも、定員は正しく制御されます。
- **発表者限定** — リンクされた Conference Program アクティビティで
  採択された発表の発表者であるユーザーのみが購入・取得できるように制
  限します（詳しくは後述）。
- **有効期間（開始／終了）**（任意）— 例えば「1日限定チケット」を特
  定の日のみ有効にする、といった利用制限を設定できます。
- **公開／非公開** — 削除せずに、購入ページからそのチケット種別を一時
  的に非表示にできます（完売した場合や販売期間終了後などに便利で
  す）。

### 発表者限定チケット

**発表者限定**に設定されたチケット種別は、購入・取得しようとした時点
で、そのユーザーが（名前ではなく Moodle アカウントで判定して）リンク
先の Conference Program アクティビティで**採択済み**の発表の発表者と
して登録されているかどうかをチェックします。この判定は、Conference
Program が公開フェーズ（Display phase）に切り替わっているかどうかに
関係なく行われます。つまり発表者は、公開スケジュールが公開される前で
も、採択が決まった時点ですぐに発表者用チケットを取得できます。

### グループ・登録方法によるチケットの自動付与

チケット種別を追加・編集する際の**「Auto-grant」**セクションでは、そ
のチケット種別を、このコースに既に設定されているコースグループ**また
は**特定の登録方法（両方は選択不可）にリンクできます。以後、そのグ
ループに参加した、またはその登録方法で登録されたユーザーには、通常の
価格に関係なく、この種別の無料チケットが自動的に発行されます。例え
ば「ボランティア」グループへのチケット提供や、特定のセルフ登録キーで
の登録者全員への提供などに便利です。

このリンクを保存すると、今後参加・登録するユーザーだけでなく、現在既
にグループに所属している、または登録されている全員にも直ちにチケッ
トが付与されます。

後でユーザーがグループを抜けたり登録を解除されたりしても、そのチケッ
トは**そのまま残ります**——自動的に取り消されることはありません。付与
条件が失われたチケットを確認し、必要であれば手動で取り消すには、チ
ケット種別管理画面から**「Orphaned tickets」**を開いてください。そこ
には、そのようなチケットが、孤立した理由（グループを離脱／登録解除）
や、既にチェックイン済みかどうかとともに一覧表示され、**「Revoke」**
操作でチケット（および記録済みのチェックイン）を完全に削除し、定員を
再び解放できます。

### グループ・登録方法を購入資格とする

チケット種別の**「Eligibility（購入資格）」**セクションは、上の
Auto-grant とは別の設定です。どちらもコースグループ・登録方法にリン
クするという点で似ているため混同しやすいですが、違いは次のとおりで
す。

- **Auto-grant（自動付与）** は、そのグループに参加した、またはその
  登録方法で登録された瞬間に、無料チケットを**自動的に発行**します。
- **Eligibility（購入資格）** は逆に、そのチケット種別を購入・取得で
  きる**資格があるかどうか**だけを判定します。資格を満たしていても、
  購入・取得の操作自体は本人が通常どおり行う必要があります。

購入資格用のグループまたは登録方法を設定すると（Auto-grant と同様、
両方の同時設定はできません）、そのグループのメンバー、またはその登録
方法で登録されたユーザーだけがこのチケット種別を購入・取得できるよう
になります。条件を満たさないユーザーには、そもそもこのチケット種別が
表示されません。この設定は上記の**発表者限定**の制限と併用でき（置き
換えるものではありません）、1つのチケット種別に対して両方・どちらか
一方・どちらも設定しない、のいずれも選べます。

### プロモコード

**「Manage promo codes」** を開くと、redeem（利用）することで指定した
チケット種別を無料で取得できるコードを作成できます。通常の価格に関係
なく無料になります。各コードについて、対象のチケット種別、利用回数の
上限（任意）、有効期限（任意）を設定できます。プロモコードで取得した
チケットには、決済が発生していないため領収書は発行されません。

### 名札・チケット・領収書・証明書のテンプレート

**「Manage templates」** を開くと、リッチテキストエディタ（TinyMCE）
を使って、次の4種類の文書をそれぞれ個別にデザインできます。各テンプ
レートは、自由に書き換えられるシンプルな初期レイアウトがあらかじめ入
力された状態で開きます。テンプレート内には、以下のプレースホルダーを
自由な位置に挿入でき、文書の生成時に実際の値へ置き換えられます（下記
の `[[like this]]` という記号は、このサイトで現在設定されている区切り
記号です——詳しくは
[プレースホルダーの区切り記号の変更](#プレースホルダーの区切り記号の変更)
を参照してください）。

- `[[fullname]]`、`[[email]]` — 参加者の氏名とメールアドレス。
- `[[tickettype]]` — チケット種別の名前。
- `[[confcheckinname]]` — このアクティビティ自体の名前。
- `[[coursefullname]]`、`[[courseshortname]]` — このアクティビティが
  属するコースの正式名称と短縮名。
- `[[origin]]` — チケットの取得方法（購入・無料・プロモコード・自動付
  与）。
- `[[qrcode]]` — 参加者固有のQRコード画像。
- `[[submissiontitle]]`、`[[track]]` — 参加者自身の採択済み発表のタイ
  トルとトラック。ただし（Conference Program とのリンクを通じて）対
  象資格のある発表者である場合**のみ**表示され、それ以外の参加者では
  空欄になります。また、複数の発表を持つ発表者であっても、常に**最初
  の**採択済み発表のみが表示されます。
- `[[presentationinfo]]` — 上記と似ていますが、対象資格のある発表者が
  行う**すべて**の採択済み発表を（最初の1件だけでなく）1行ずつ列挙し
  ます。各行に何を表示するかは、下記4種類のテンプレートごとに個別に
  設定できます——詳しくは
  [presentationinfo に表示する内容のカスタマイズ](#presentationinfo-に表示する内容のカスタマイズ)
  を参照してください。

入力ミスや、条件に合わないプレースホルダーは、文書の生成時に単純に削
除されるだけで、区切り記号付きの文字列がそのまま表示されることはあり
ません。

- **名札（Badge）** — 印刷用の名札。参加者ごとに固有のQRコードが含ま
  れます。
- **チケット（Ticket）** — 購入確認・入場用のパスです。
- **領収書（Receipt）** — 実際に決済が行われた有償チケットに対しての
  み発行されます。無料チケット、プロモコード、自動付与によるチケット
  には発行されません。
- **証明書（Certificate）** — 参加証明書です。参加者が受付でチェック
  インを済ませた後にのみ発行されます（後述）。

### presentationinfo に表示する内容のカスタマイズ

4種類のテンプレートにはそれぞれ、コンテンツエディタのすぐ下に
**「Presentation info format」** という入力欄があります。これは「テ
ンプレートの中のさらに小さなテンプレート」で、ここに入力した内容が
発表者の発表**ごとに**1回ずつ適用され、その結果が改行でつなぎ合わさ
れて `[[presentationinfo]]` が作られます。

ここでは専用の小さな記号セット——`{title}` と `{track}`（上記の
`[[ ]]` 形式の記号とは異なる、意図的に単一の中括弧）——が使えます。
例:

```
<strong>{title}</strong>（{track}）
```

空欄のままにすると、タイトルのみが表示されます。トラックが設定され
ていない発表では `{track}` は空欄になるため、その周りに入力した文字
（上記の括弧など）はその発表について空のまま表示されます。

### プレースホルダーの区切り記号の変更

デフォルトでは、テンプレートのプレースホルダーは二重角括弧で記述しま
す（例：`[[fullname]]`）。サイト管理者は、このアクティビティの管理設
定（サイト管理 → プラグイン → 活動モジュール → Conference Check-in）
から、サイト全体の開き／閉じ区切り記号を変更できます。この設定はアク
ティビティ単位ではなくサイト全体に適用されるため、すべての主催者が同
じ規則を共有できます。

この設定をテンプレート作成後に変更すると、既存のテンプレートは古い区
切り記号のままの文字列を使い続けることになり、それらは認識されなくな
ります——その場合は、既存のテンプレートを新しい区切り記号に合わせて自
分で更新する必要があります。

### 名札・チケット・領収書の一括ダウンロード

一括ダウンロード権限を持っている場合、アクティビティのメイン画面にあ
る **「Download all Badges」**／**「Tickets」**／**「Receipts」** か
ら、その種類の参加者全員分のPDFをまとめた1つのZIPファイルをダウン
ロードできます。削除済みユーザーのチケットや、チケット種別自体が削除
されている場合は、ダウンロード全体を失敗させることなくスキップされま
す。領収書のZIPには、実際に決済が行われたチケットのみが含まれます。

### 参加者の受付（スキャン）

イベント当日は、チェックイン権限を持つスタッフがアクティビティのメイ
ン画面から**「Scan check-in」**を開き（Moodle アプリ内でも同じ画面が
モバイル向けアドオンとして利用可能）、参加者が到着するたびにチェック
インを記録します。スキャナーには常にテキスト入力欄が用意されており、
USB接続やBluetooth接続のバーコードスキャナー（キーボード入力として認
識されるタイプ）からの入力を受け付けます——名札をスキャンするだけで即
座にチェックインが記録されます。ブラウザがカメラによるスキャンに対応
している端末では、**「Scan with camera」**ボタンも表示され、スマート
フォンやタブレットのカメラを名札に直接向けてスキャンすることもできま
す。既にチェックイン済みの名札を再度スキャンした場合は「既にチェック
イン済み」と表示され、重複した記録が作られることはありません。また、
別のイベント用の名札をスキャンした場合は、その旨が明確に表示され拒否
されます。

## 参加者向け

### チケットの購入・取得

Conference Check-in アクティビティを開き、利用可能なチケット種別から
選択します。価格が0のチケット種別、または有効なプロモコードを利用す
る場合は、決済手続きなしですぐにチケットが発行されます。有償のチケッ
ト種別の場合は、サイトの通常の決済手続きに進みます。同じページの
**「Your tickets」** には、これまでに取得したすべてのチケットが、それ
ぞれのダウンロードリンクとともに一覧表示されます。

一部のチケット種別は、発表者限定、または特定のコースグループ・特定の
登録方法のメンバー限定に設定されている場合があります。いずれかの条件
を満たしていない場合、そのチケット種別はそもそも選択肢として表示され
ません。

### 名札とQRコード

**「Your tickets」** から、名札（さらに希望すればチケットも、実際に
決済が行われたチケットのみ領収書も）をPDFとしてダウンロードできま
す。名札にはチェックイン時にチケットを識別する固有のQRコードが含まれ
ています。スマートフォンですぐに表示できるようにしておくか、事前に印
刷しておいてください。

### 受付でのチェックイン

会場では、名札のQRコードをスタッフに提示してスキャンしてもらいます。
イベントごとに1回行えば十分です——再度スキャンされても問題はなく、重
複した記録が作られることはありません。

### 参加証明書のダウンロード

受付でチェックインが完了すると、アクティビティのメイン画面にある
**「Your tickets」**から参加証明書をダウンロードできるようになりま
す。チェックイン前はダウンロードできません——その場合はまだ準備がで
きていない旨が表示されます。

## よくある質問

**無料チケットでも領収書はもらえますか？**
いいえ。領収書は実際に決済が行われた有償チケットに対してのみ発行され
ます。無料チケットやプロモコードによるチケットには、そもそも発行すべ
き決済記録がありません。

**採択された発表者ですが、発表者用チケットを取得するのに、スケジュー
ルが公開されるまで待つ必要がありますか？**
いいえ。発表者資格の判定は、ご自身の採択状況に基づいて直接行われるた
め、Conference Program の公開フェーズ（Display phase）への切り替えを
待つ必要はありません。

**表示されるはずのチケット種別が表示されません——なぜですか？**
発表者限定に設定されているか、特定のコースグループまたは特定の登録方
法のメンバーに限定されている可能性があります。これらは主催者がチケッ
ト種別ごとに設定できる、互いに独立した2つの条件です（両方・どちらか
一方・どちらも設定しない、のいずれもあり得ます）。自分に資格があるは
ずだと思われる場合は、主催者にご確認ください。

**「バッジ（badge）」は Moodle の Open Badge と同じものですか？**
いいえ。このアクティビティでの「バッジ」は、常にイベント当日に身につ
ける印刷用の名札を指しており、Moodle 本体のデジタル Open Badges（達
成バッジ）機能とは関係ありません。

**チケットが自動付与されたグループを抜けたら、チケットは失われます
か？**
いいえ。一度付与されたチケットが自動的に取り消されることはありませ
ん。主催者は「Orphaned tickets」レポートでそのチケットの付与条件が失
われたことを確認し、手動で取り消すことができますが、自動的には何も起
こりません。

**チェックインする前に証明書を取得できますか？**
いいえ——証明書は、実際に会場でチェックインが完了した後にのみ生成され
ます。それより前にダウンロードしようとすると、まだ準備ができていない
旨が表示されます。
