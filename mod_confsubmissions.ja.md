# Conference Submissions（発表投稿）— ユーザーマニュアル

**アクティビティタイプ:** `mod_confsubmissions`
**Conference Tools スイートの一部です** ([概要](README.md))

Conference Submissions は「発表募集（Call for Abstracts）」用のアクティビ
ティです。コースに追加することで、発表者がトーク・セッションの提案を投
稿できるようになります。文字数制限、トラック（分野）分け、独自項目の追
加などを自由に設定できます。ここで投稿された内容は、そのまま
[Conference Program](mod_confprogram.ja.md) の審査プロセスに引き継がれ
ます。

> 画面上の項目名やボタンの表示は英語のままです（本マニュアルは操作の説
> 明を日本語で行うものです）。

## 主催者向け

### アクティビティの追加

1. コースの編集モードをオンにし、**「活動またはリソースを追加する」**
   を選択します。
2. **Conference Submissions** を選びます。
3. 名前（例：「発表募集」）を入力し、必要であれば発表者向けの説明文も
   入力します。
4. **Submission limits（投稿の制限）** で、タイトルとアブストラクトそ
   れぞれについて、最大語数・最大文字数を個別に設定します（どちらも
   任意項目で、0のままにするとその制限は無効になります）。両方を設定
   すると、どちらか一方でも超えた時点で投稿できなくなります。これは
   英語と日本語が混在する内容に便利です（例：英語話者向けの語数制限
   と、全角の日本語テキスト向けの文字数制限を、同じ項目に同時に適用
   する）。発表者が入力中は設定されている制限に応じてリアルタイムで
   カウンターが表示され、いずれかの制限を超えると投稿できません。
5. **Call dates（募集期間）** で、投稿の受付開始日時・締切日時を設定し
   ます。開始日より前や締切日を過ぎた後は新規投稿を開始できず、締切日
   を過ぎると既存の投稿も編集できなくなります。
6. 保存してコースに戻ります。

### トラックの管理

トラックは投稿をテーマごとに分類するためのものです（例：「教育論」
「技術」「事例紹介」など）。アクティビティを開き、**「Manage
tracks」**（編集教師・管理者のみ表示）を選択します。

- **トラックの追加**：名前を付け、任意で**色**とあらかじめ用意された
  アイコンセットから**アイコン**を選べます。ここで選んだ色とアイコン
  は、このアクティビティ内だけの装飾ではありません。Conference
  Program の採択済み投稿一覧や、Conference Scheduler のブロックスケ
  ジュールなど、スイート全体でこのトラックが表示される場所すべてで、
  色付きのピル（バッジ）として表示されます。複数のトラックがある場合
  は、見分けやすい色を選ぶとよいでしょう。
- **トラックの編集・削除**：名前・色・アイコンはいつでも変更できま
  す。既に投稿で使用されているトラックを削除しようとすると確認が表示
  されます。削除後も、既存の投稿にはトラック名のテキスト記録は残りま
  すが、（削除済みの）トラックそのものへのリンクは失われます。
- 発表者は投稿時にこの一覧からトラックを選択します。トラックが1つも
  作成されていない場合、投稿フォームにトラックの項目自体が表示されま
  せん。

### 発表タイプの管理

発表タイプ（例：「ライトニングトーク」「ワークショップ」「基調講演」）
を使うと、発表者が自分の提案するセッションの種類を選択でき、タイプご
とに標準的な発表時間（分）をあらかじめ設定できます。**「Manage
submission types」** を開いて、タイプの追加・名称変更・削除を行いま
す。

発表タイプを1つでも追加すると、発表者は投稿時に必ずいずれかのタイプ
を選択する必要があります。発表タイプを1つも設定していないインスタン
スでは、この項目自体が表示されないため、よりシンプルな発表募集にした
い場合はこの機能を使わなくてもかまいません。選択された発表タイプの所
要時間は、[Conference Scheduler](mod_confscheduler.ja.md) がその投稿
をスケジュールに配置する際の初期のブロック長として使用されます（配置
後も自由にサイズ変更でき、その後にタイプの所要時間設定を変更しても既
存のブロックには影響しません）。

### 独自項目（カスタムフィールド）の設定

タイトル・アブストラクト・トラック以外にも、発表者への質問を独自に追
加できます。例えば「希望セッション時間」「想定する聴講者レベル」「必
要な機材」などです。**「Manage custom fields」** を開いて項目の追加・
並べ替え・削除を行います。

各項目について、以下を選択します。

- **項目名**：投稿フォーム上に質問文として表示されます。
- **項目タイプ**：テキスト（1行）、テキストエリア（複数行）、チェッ
  クボックス、日付、ドロップダウン（メニュー、選択肢は自由に設定可
  能）、数値、URL から選べます。ドロップダウンにすれば回答が統一され、
  後で絞り込みしやすくなります。自由記述にしたい場合はテキストエリア
  が向いています。**回答が1件でも提出された項目は、タイプを変更でき
  なくなります**（保存済みの回答は取得時のタイプでのみ意味を持つた
  め）。タイプを変えたい場合は、項目を削除（回答も削除されます）して
  から新しい項目を追加してください。
- **必須項目かどうか**：投稿前に必ず入力させるかどうかを設定します。

ここで追加した項目は、設定した順番どおりに投稿フォームへ表示され、そ
の回答は主催者・審査者が投稿内容を確認する際に他の項目と並んで表示さ
れます。

### 開催日程と希望日程

アクティビティの設定で、任意項目として**開催開始日時**と**開催終了日
時**を設定できます。この2つは必須ではなく、それ自体が他の動作を制限
することもありません。目的は、その下にある別の設定「**Offer
preferred dates（希望日程を提示する）**」が使用する日程範囲を決めるこ
とだけです。

**Offer preferred dates** を有効にすると（開催開始・終了日時の両方が
設定されている場合に有効化できます）、投稿フォームに開催日ごとのチェッ
クボックスが表示され、既定ではすべてチェックされています。発表者は参
加できない日のチェックを外します。

既定では、[Conference Scheduler](mod_confscheduler.ja.md) の自動スケ
ジューラーはその発表をチェックされた日にのみ配置します（時間帯は自由
に選ばれます）。チェックされた日に空きがない場合、発表者が除外した日に
移す代わりに未配置のまま残します。ただし主催者は、自動スケジューラー実
行時に **「希望日程を無視する」** をチェックすれば、その1回に限りどの
日にもフォールバックできます。編集モードで特定の日を選択すると、その
日を選んでいない未配置の発表は非表示になるため、参加できない日に誤っ
てドラッグすることを防げます。

**特定の日を無効にする**：例えばワークショップ専用の日や、多くの発表
者が参加できないとわかっている日など、誰にも希望日として提示したくな
い日がある場合は、**「Manage disabled preferred days」** を開いてその
日にチェックを入れます。一般の発表者から見ると、無効にした日のチェッ
クボックス自体は投稿フォーム上に引き続き表示されます（誰から見ても同
じ日程リストになります）が、灰色表示になりチェックできなくなります。
無効化する際に理由（例:「会場が使用不可」）を任意で入力でき、設定す
るとその理由が投稿フォーム上でグレーアウトされた日の横に括弧書きで表
示され、発表者に理由が伝わります。編集教師・管理者はこの設定の影響を
受けず、代理で投稿・編集する際にも無効化の有無にかかわらずすべての日
を選択できます。

### すべての投稿の確認

編集教師・管理者は、**「All submissions」** から自分以外も含めたすべ
ての投稿を確認できます。タイトル・トラック・現在のステータス・発表者
が一覧表示され、各投稿の詳細にもリンクしています。実際の審査や採否の
決定は、リンクされた Conference Program アクティビティ側で行います。
こちらの画面は、あくまで届いた投稿全体を把握するためのものです。

サイト管理者およびマネージャーロールを持つユーザーは、この画面から投
稿を完全に**削除**することもできます。発表者・回答内容も含めて完全に
削除され、元に戻すことはできません。編集教師にはこの削除権限は意図的
に付与されていません（投稿を能動的に取り下げられるのは投稿者自身の
「Withdraw」操作のみで、完全な削除はマネージャー・管理者のみが行えま
す）。

編集教師・管理者は、この画面から任意の投稿を**編集**することもできま
す。各行に表示される「Edit」リンクから、投稿者本人が使うものと同じ
フォームが開きます。審査後にトラックを修正したり、誤字を直したりする
際に便利です。募集期間が終了していても、他人の投稿を代理編集する場合
は上記の受付開始・終了日時による制限を受けません（投稿者本人が自分の
投稿を編集する場合のみ、この制限が適用されます）。他の発表者の投稿を
編集している間は、誰の投稿を編集しているかを示すバナーが表示されま
す。同じ「Edit」リンクは Conference Program の Decision report にも表
示されるため、審査画面を離れずにトラックを修正できます。また、各投
稿の詳細ページにも表示されます。

取り下げ済みの投稿には、この画面に**「Unwithdraw（取り下げ解除）」**
リンクが表示されます（編集教師・管理者のみ）。取り下げ解除を行うと、
単に「提出済み」に戻すのではなく、取り下げられる直前に持っていたステー
タス（採択・却下・提出済み）が復元されます。そのため、すでに採択され
ていた発表を取り下げた発表者が気持ちを変えた場合でも、その採択の記録
が失われることはありません。

### 多言語対応コンテンツ

トラック名・発表タイプ名・独自項目（カスタムフィールド）名・希望日程
無効化時の理由は、いずれも Moodle の**多言語コンテンツ（Multi-Language
Content）フィルター**に対応しています（サイトで有効になっている場合）。
名前や理由を入力する際、各言語の内容を
`<span lang="xx" class="multilang">...</span>` タグで囲む（言語ごとに
1つの span）ことで、閲覧者にはそれぞれの言語設定に合った内容のみが表
示されます。

### 通知

**新規に追加したアクティビティでは通知は初期状態でオフ**になっていま
す。これらのメールを送信したい場合は、下記の **「Enable
notifications」** をオンにしてください。

投稿が行われると、投稿者だけでなくすべての発表者（Moodle アカウントを
持つ場合）に自動でメールが送信されます。また、投稿が取り下げられた場
合は、「任意の投稿を編集する」権限を持つ主催者全員（既定では編集教師
とマネージャー）にメールで通知されます。どちらも即座に送信されます
——Conference Program や Conference Scheduler とは異なり、ここには送
信待ちキューはありません。いずれも一度きりのイベントであり、まとめて
送信する利点がないためです。

**「Manage notifications」** を開くと、それぞれの通知の件名・本文をカ
スタマイズできるほか、**「Enable notifications」** のチェックを外すこ
とで、このアクティビティの通知を完全に無効化できます（発表者への連絡
を別の手段で行いたい場合などに便利です）。独自のテンプレートを保存す
るまでは組み込みの既定文面が使用され、件名・本文のいずれかを空欄で保
存した場合も、空のメールではなく既定の文面に戻ります。本文には
`[[fullname]]`、`[[submissiontitle]]`、`[[coursename]]` などのプレース
ホルダーを使用でき、通知種別ごとに使用できる項目の一覧は画面上に表示
されます。

## 発表者向け

### 発表概要（アブストラクト）の投稿

1. コース内の Conference Submissions アクティビティを開きます。
2. **「Submit an abstract」**（募集期間中のみ表示）を選択します。
3. **タイトル**と**アブストラクト**を入力します。文字数・語数の制限が
   設定されている場合はカウンターを確認しながら入力してください。
4. 主催者がトラックを設定している場合は、**トラック**を選択します。
5. 主催者が追加した質問項目があれば入力します。
6. **Speakers（発表者）** セクション（後述）を確認し、投稿します。

### 希望日程の選択

主催者が **Offer preferred dates（希望日程を提示する）** を有効にして
いる場合（上記の主催者向けセクションを参照）、開催期間中の日付ごとに
チェックボックスが表示され、デフォルトですべてチェックされていま
す。参加できない日のチェックを外してください——自動スケジューラー
は、チェックを残した日のいずれかに発表を配置しようとしますが、これは
希望であり保証ではありません。主催者が特定の日を無効にしている場合、
その日はリストには表示されますが、灰色になっており選択できません——
理由が設定されていれば、その日の横に括弧書きで表示されます。

### 共同発表者の追加

Speakers セクションには、投稿者自身が最初から「Speaker 1」として表示
されます。共同発表者を追加するには **「Add speaker」** を選び、以下の
どちらかの方法で入力します。

- **登録済みユーザーを検索**：名前を入力し始めると、そのコースに登録
  済みのユーザーのみが候補として表示されます。選択すると、（ユーザー
  IDではなく）正しくフルネームが表示されます。
- **手動で入力**：コースに登録されていない共同発表者（外部ゲスト講演
  者など）の場合、名前とメールアドレスを直接入力します。後で投稿を編
  集する際も、この行が「手動入力」として設定されていたことをフォーム
  が記憶しているため、再入力したり内容が消えたりする心配はありませ
  ん。

主催者が許可する範囲で共同発表者を何人でも追加でき、誤って追加した行
は削除できます。また、リストは**ドラッグして並べ替え**が可能です。こ
こで設定した順番が、投稿が表示されるあらゆる場所での発表者の表示順に
なります。

### 投稿の編集・取り下げ・削除

発表募集の受付期間中は、**「My submissions」** から自分の投稿を選び、
発表者リストとその順番、希望日程も含めて自由に編集できます。締切日を
過ぎると投稿は編集不可（読み取り専用）になります。締切後に変更が必要
な場合は主催者に直接連絡してください。

参加できなくなった場合は、編集する代わりに **「My submissions」** か
ら **「Withdraw（取り下げ）」** を選んでください（確認画面が表示され
ます）。これは投稿を削除するのではなく「取り下げ済み」のステータスに
変更するものなので、主催者側には記録が残ります。取り下げと削除は別の
操作です。投稿を完全に削除できるのはサイト管理者またはマネージャーの
みです。

## よくある質問

**アブストラクトは複数投稿できますか？**
はい。主催者からシステム外で別途案内がない限り、1人あたりの投稿数に
制限はありません。

**トラックの選択項目が表示されません**
主催者がこの募集用にまだトラックを設定していないか、項目自体を無効に
しています。トラックの設定は任意です。

**共同発表者の行を誤って「手動入力」にしてしまいました。検索モードに
戻せますか？**
はい。投稿を編集画面で開き、その行のモードを変更してください。投稿を
保存すると変更が反映されます。

**投稿した後はどうなりますか？**
投稿を受け付けたことを知らせるメールが届きます（主催者が通知を無効に
していない限り）。その後、リンクされている Conference Program アクティ
ビティの審査プロセスに引き継がれます。主催者がプログラムを公開すると、
採択・却下・補欠のいずれかの判定結果が決まった時点で、再度メールが届
きます。

**誤って希望日程のチェックをすべて外してしまいました。どうなります
か？**
希望日程が一つも設定されていない状態は、そもそも希望を登録していない
状態と同じ扱いになります。自動スケジューラーはどの日にでも配置できま
す。「どの日も参加できない」という意味にはなりません。

**希望日程のチェックボックスの一つが灰色になっていて選択できません**
主催者がその特定の日を希望として提示しないよう設定しています。これは
その日にスケジュールされないという意味ではなく、希望として申告できな
いという意味です。理由が設定されていれば、その日の横に括弧書きで表示
されます。その日に特にスケジュールしてほしい場合は、主催者に直接連絡
してください。
